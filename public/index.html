How Encryption Works

Two independent layers of AES-256-GCM protection

1

Key Derivation

Your passphrase is run through PBKDF2 (100,000 iterations, SHA-256) to derive a strong 256-bit AES key — entirely in your browser.

→
2

Client Encryption

Your file or text is encrypted with AES-256-GCM using the Web Crypto API before a single byte is sent to the server.

→
3

Server Encryption

The already-encrypted blob receives a second AES-256-GCM layer using a secret server key. Two independent keys protect your data.

→
4

Secure Storage

Only the doubly-encrypted ciphertext is saved to disk — linked to your account. No plaintext, no readable keys.
